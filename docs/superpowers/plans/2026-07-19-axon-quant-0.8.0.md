#### 2.3 B3 跨 instrument gamma 协方差

**目标**:跨 instrument 计算 gamma 协方差矩阵,支持 portfolio-level gamma 风险聚合(非对角项)。

**0.7.0 范围**:

- `PortfolioRiskEngine` 只算 per-leg gamma(对角矩阵:`HashMap<Instrument, f64>`)
- portfolio_gamma = Σ per-leg gamma(忽略跨 instrument 相关性)
- 多 leg 高相关组合(如 BTC spot + BTC perp)重复计入 gamma 风险,实际 portfolio gamma 应折扣

**0.8.0 B3 目标**:

返回完整 n×n 协方差矩阵(对称半正定):

```rust
/// 返回所有持仓 instrument 两两间的 gamma 协方差
/// 返回 HashMap<(Instrument, Instrument), f64>:
/// - 对角 (i, i) = var(returns_i) × qty_i² × contract_size_i²
/// - 非对角 (i, j) = cov(returns_i, returns_j) × qty_i × qty_j × contract_size_i × contract_size_j
pub fn gamma_covariance_matrix(&self, portfolio: &Portfolio) -> HashMap<(Instrument, Instrument), f64>
```

**关键设计:Ledoit-Wolf 收缩**

- 样本协方差矩阵 Σ̂ 在 N << p(N=样本数,p=instrument 数)时高度奇异
- 经典 Ledoit-Wolf (2004) 收缩:`Σ_shrunk = α·F + (1-α)·Σ̂`,其中 F = `μ·I`(标量目标)
  - α 自动选择,优化 MSE(`α* = min(1, (Σ(π̂_ii²) - π̄²·Σ(p̂_ij²)) / (Σ(Σ̂_ij²) - π̄²·Σ(p̂_ij²))`)
  - 在工程上用简化版:`α = min(1, shrinkage_target / shrinkage_sample)`,或简化为常量 0.2
- 0.8.0 B3 采用 **简化 Ledoit-Wolf 收缩**(constant 0.2 标量目标 = `trace(Σ̂) / p · I`):
  - `Σ_shrunk = (1-α)·Σ̂ + α·F`,α = 0.2 默认,F = `mean_var · I`
  - 保证正定性(加 α·F 后特征值最小为 α·mean_var > 0)
  - 工程实现简单,无外部依赖

**步骤分解**:

- [x] **B3.1** `MarketDataSource` 扩展 `mark_returns(instrument, lookback) -> Vec<f64>`
  - 默认实现:`let r = mark_history(...).windows(2).map(|w| w[1].1 / w[0].1 - 1.0).collect()`
  - `InMemoryMarketData` / `CsvMarketData` 自动继承
  - `latest_returns` 便捷接口(`mark_returns(.., n).last()`)
- [x] **B3.2** 私有 helper `ledoit_wolf_shrink(returns_matrix: &[Vec<f64>], alpha: f64) -> Vec<Vec<f64>>`
  - 输入:`N×p` returns 矩阵(N=样本,p=instrument)
  - 步骤:
    1. 计算样本协方差 Σ̂(N×N)用 N-1 无偏估计
    2. 计算 target F = `trace(Σ̂) / p · I_p`(标量对角)
    3. 返回 `(1-α)·Σ̂ + α·F`
  - 数值稳定:用 `f64::EPSILON * p` 防 0/0
  - 单测:常对角(returns 全相等)→ Σ̂ = 0 → shrunk = `α·F` > 0(正定)
  - 单测:正交 returns(独立)→ Σ̂ 对角 → shrunk 对角
- [x] **B3.3** `PortfolioRiskEngine::gamma_covariance_matrix(portfolio) -> HashMap<(Instrument, Instrument), f64>`
  - 收集所有有持仓的 instrument → list `insts: Vec<Instrument>`
  - 取每 instrument 的 `mark_returns(inst, lookback)` → `N×p` returns
  - N < 2 或 p == 0 → 返回空 HashMap(0.7.0 兼容行为)
  - N < p → 触发 Ledoit-Wolf 收缩(避免 singular)
  - N >= p → 直接用样本协方差
  - 用 `contract_size_of` 缩放每行:`cov_scaled[i][j] = cov_returns[i][j] * qty[i] * qty[j] * cs[i] * cs[j]`
  - 返回 `HashMap<(Instrument, Instrument), f64>` 形式(只存上三角 `i <= j`,长度 `p(p+1)/2`)
  - **存储约定**:`HashMap` 中 `(a, b)` 顺序应与 `insts` 顺序一致;调用方需要矩阵对称性时用 `a < b` 顺序访问
- [x] **B3.4** `PortfolioRiskEngine::portfolio_gamma_with_covariance(portfolio) -> f64`
  - 标准组合 gamma 公式:`Σ_i γ_i² + 2·Σ_{i<j} γ_i·γ_j·ρ_{ij}`(实际即矩阵自求和)

### Phase 3:撮合层重写(预计 4-5 周)

**目标**:L3 matching engine 完整重写,性能 gate < 2x vs L2

#### 3.1 A1 L3MatchingEngine 完整重写

**0.7.0 现状**:L3Book 只是 L2 引擎的"稳定只读视图",撮合本体还是 L2

**0.8.0 方案**:

- 新增 `L3MatchingEngine`(独立 struct,不复用 L2)
  - `BTreeMap<Price, Vec<L3Order>>` bids + asks
  - `submit(order)`:price-time priority,partial fill 跟踪到 `L3Order.filled_qty`
  - `cancel(order_id)`:从 book 移除
  - `best_bid()` / `best_ask()` / `depth(levels)`
- `L3Order { order_id, side, qty, filled_qty, timestamp_ns, source }`
- `L3Fill` 跟踪每笔 partial fill 的"被吃方"
- `L3Book::from_l3_engine()` 工厂

**性能 gate**:

- [ ] bench 脚本:`benches/l3_vs_l2.rs` 同 case 对比
- [ ] gate:L3 撮合 ≤ 2x L2 时间 + ≤ 2x 内存
- [ ] 不达 gate → 降级为 `l3-matching` feature flag,默认 L2

#### 3.2 A2 撮合层多态扩展

- [ ] `Strategy` trait:

  ```rust
  pub trait Strategy: Send {
      fn on_fill(&mut self, fill: &FillRecord, ctx: &mut StrategyContext) -> Vec<Order>;
      fn on_bar(&mut self, bar: &Bar, ctx: &mut StrategyContext) -> Vec<Order>;
  }
  ```

- [ ] `MultiAssetStrategy` trait:多 leg 协同决策
- [ ] `EngineRouter`:根据 `Strategy` 路由到 `Box<dyn MatchingEngine>`
- [ ] 测试:`Box<dyn Strategy>` + `Box<dyn MatchingEngine>` 联动

#### 3.3 A3 撮合 hot-path 性能优化

**0.7.0 bench 测得**:`inner.submit` ~150µs 占主导,`compute_impact` ~100ns

**优化方向**:

- [ ] **SIMD 价格比较**:`f64` 价格排序用 `packed_simd` 或 `std::simd`(MSRV 检查)
- [ ] **Arena allocator**:每 bar 内的 fill 记录用 `Bump` allocator 一次性回收
- [ ] **订单结构紧凑**:`Order` 从 ~256B 压到 ~64B(`#[repr(C, packed)]` 谨慎使用)
- [ ] **SoA vs AoS**:`L1Book` 内部 bids/asks/timestamps 改 SoA
- [ ] bench 对比:优化前 vs 优化后,目标 `inner.submit` ≤ 50µs
