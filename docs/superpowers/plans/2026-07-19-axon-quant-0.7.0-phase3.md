## Phases

### Phase 3.1:Trait 化补齐(预计 30-45 min,1 commit)

**目标**:让 L1 / L2 / Impacted / MultiAsset 都能作为 `Box<dyn MatchingEngine>` 多态使用。

#### 3.1.1 `L2MatchingEngine` impl `MatchingEngine`

- 现成方法齐全,加 5-8 行 `impl` 块
- `submit` / `cancel` / `best_bid` / `best_ask` / `spread` / `active_order_count` 直接转发到 `self.inner`
- `depth(levels)` 用现有的 `export_entries` + slice
- `clear_book` / `clear_book_for` 转发到 `self.inner`
- `seed_liquidity(...)` 转发到 `self.inner`,instrument 参数丢弃(L2 默认所有 instrument 共享 book,实际就是单一 book)

#### 3.1.2 `ImpactedMatchingEngine` impl `MatchingEngine`

- 现成方法齐全
- `seed_liquidity` 已存在(190 行)
- `clear_book` 已存在(222 行)
- 同样加 5-8 行 `impl` 块

#### 3.1.3 `MultiAssetMatchingEngine` impl `MatchingEngine`

- ⚠️ **多 instrument 路由**:`MatchingEngine::best_bid()` 等无 instrument 参数,语义模糊
- 方案:新增 `with_primary(instrument)` 构造函数,无 instrument 方法走 primary instrument
- `submit(order) -> SubmitResult` —— 适配新签名(原 `submit -> Vec<MatchFill>`)
- `cancel(order_id) -> bool` —— 新增:全 instrument 扫 order_index 取消
- `best_bid/best_ask` —— 转发到 primary
- `active_order_count` —— 跨所有 instrument 合计
- `clear_book` —— 清所有 instrument
- `clear_book_for(instrument)` —— 单 instrument 清(0.7.0 已有 trait 方法)
- `seed_liquidity(...)` —— 注入到 primary

#### 3.1.4 测试

- `crates/axon-backtest/tests/trait_polymorphism.rs`:用 `Box<dyn MatchingEngine>` 装 L1/L2/Impacted/MultiAsset,统一调 submit,验证所有 impl 行为一致

### Phase 3.2:`L3Book` 完整 L3 可见视图(预计 30-45 min,1 commit)

**目标**:新增数据结构,作为撮合引擎的"稳定对外只读视图"。

#### 3.2.1 数据结构

- 新文件:`crates/axon-backtest/src/matching/l3/book.rs`
- `L3Book { bids: BTreeMap<Price, Vec<L3Order>>, asks: BTreeMap<Price, Vec<L3Order>> }`
- `L3Order { order_id: u64, side: Side, qty: f64, timestamp_ns: i64 }` —— 精简视图,稳定 wire format
- 不持有原 `Order`(避免状态机污染 + 字段不稳定性)
- 公共方法:
  - `best_bid() -> Option<Price>` / `best_ask() -> Option<Price>`
  - `mid_price() -> Option<f64>`
  - `spread() -> Option<Price>`
  - `total_bid_qty() -> f64` / `total_ask_qty() -> f64`
  - `orders_at(side: Side, price: Price) -> &[L3Order]`
  - `levels_bid() -> impl Iterator<Item = (Price, &[L3Order])>` / `levels_ask() -> ...`

#### 3.2.2 工厂

- `L3Book::from_l2_engine(engine: &L2MatchingEngine) -> L3Book`
- `L3Book::from_l1_engine(engine: &L1MatchingEngine) -> L3Book`
- `L3Book::from_multi_asset(engine: &MultiAssetMatchingEngine, instrument: &Instrument) -> L3Book`

#### 3.2.3 序列化(可选)

- `#[derive(Serialize, Deserialize)]` 让 `L3Book` 可 dump 到 JSON(backtest 报告用)

#### 3.2.4 测试

- `crates/axon-backtest/src/matching/l3/book.rs` 内嵌 5-6 个单测,覆盖:
  - 空 book
  - 挂 3 档 bid + 2 档 ask,验证 orders_at 返回正确 order
  - mid_price / spread 计算
  - 跨 instrument 隔离(从 MultiAsset 取 BTC book,不影响 ETH book)
  - L3Order timestamp 顺序(FIFO 在 price level 内)

### Phase 3.3:e2e + 性能 gate(预计 30 min,1 commit)

#### 3.3.1 3 个 e2e 测试(`tests/test_l3_full_visibility.rs`)

- `test_l3_book_full_visibility`:挂 5 个 order 在 3 个价位,L3Book 能查每个 order 的 id / side / qty / timestamp
- `test_l3_book_after_partial_fill`:partial fill 完成后,L3Book 还显示剩余 order(不消失)
- `test_trait_polymorphism_uniform_submit`:用 `Box<dyn MatchingEngine>` 装 4 个 impl,同一组 order 跑出等价 fills

## Risks / Open Questions

1. **`MultiAsset::with_primary` 语义** —— 用"primary instrument 路由"vs"全部 instrument 聚合"vs"trait 升级为带 instrument 参数"。已选方案 A(最简),不破坏现有 caller。
2. **L3Book 是否要写库** —— 不,作为 `axon-backtest` 内部视图(0.7.0 阶段);若未来要做 OMS 风控 / 跟单回放,可独立发版。
3. **性能 gate 阈值** —— plan 3.5 写"必须与 L2 同 case 跑 < 2x 慢",用 wallclock 简单测。如果 MultiAsset 因为 instrument 路由 HashMap lookup 比 L2 慢 > 2x,需要优化(可能改成 flat array 索引)。
