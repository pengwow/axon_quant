### Phase 3.1 — A2.1 EngineRouter 抽象(预计 1 周) ✅ 2026-07-21

**目标**:把 L1 / L2 / L3 / Impacted 多引擎统一到一个 `EngineRouter` 抽象。

**最终实现**(commit 待 push,working tree):

```rust
pub enum RoutedEngine {
    L1(L1MatchingEngine),
    L2(L2MatchingEngine),
    L3(MultiAssetMatchingEngine),
    Impacted(ImpactedMatchingEngine),
}

pub enum RoutingStrategy {
    StrictByInstrument,        // 未注册 → SubmitResult::empty
    #[default]
    PerInstrumentWithFallback, // 未注册 → primary
}

pub struct EngineRouter {
    routes: HashMap<Instrument, RoutedEngine>,
    primary: Option<RoutedEngine>,
    strategy: RoutingStrategy,
}
```

**关键设计点**:

- `Send + Sync` 编译期 `const _` 断言 + 单测验证

### Phase 3.2 — A1.1 L3 单 asset 路径重写(预计 2 周)

**目标**:`MultiAssetMatchingEngine` 内部单 asset 路径的 price-time priority 完整重写。

**现状**(A3.0 baseline 后)::

- L3 透传到 L2(`L2MatchingEngine::inner.submit`)
- L2 又透传到 L1(`L1MatchingEngine::submit`)
- 真正做撮合的是 L1,L2/L3 加 index / 路由 / 暗池 / 拍卖

**问题**:
- L1 的 price-time priority 假设是单 instrument / 单 book,L3 多 asset 后多 book 的 partial fill tracking 散在 L1 内部
- L3 的 `PartialFillTracker` 仅是 wrapper,不维护 per-fill 元数据
- 套利 / 暗池 / 拍卖的 order 生命周期追踪与 L2 的 order_index 重复

**重写方向**:
- `L3CoreMatchingEngine`:新单 asset 撮合,显式 price-time priority + FIFO 队列
- `PartialFillTracker`:per-fill 元数据(fill_id, taker_order_id, maker_order_id, qty, price, timestamp) + `FillState` 状态机
- `OrderBook`:价位 → FIFO 队列(当前 L1 `BTreeMap<Price, VecDeque<Order>>` 不变,只增强 wrapping)

**最终实现**(2026-07-22):

```rust
// axon-core
pub struct FillRecord {
    pub fill_id: u64,
    pub taker_order_id: OrderId,
    pub maker_order_id: OrderId,
    pub price: Price,
    pub quantity: Quantity,
    pub timestamp: Timestamp,
    #[serde(default)]
    pub state: FillState,
}

pub enum FillState {
    #[default] Active,
    Filled,
    CancelledAfterPartial,
    CancelledNoFill,
}

// axon-backtest
pub struct PartialFillTracker {
    chain: HashMap<OrderId, Vec<FillRecord>>,
    order_instrument: HashMap<OrderId, Instrument>,
}

impl PartialFillTracker {
    pub fn track_fill(&mut self, fill: &MatchFill, instrument: &Instrument);
    pub fn mark_filled(&mut self, order_id: OrderId);
    pub fn mark_cancelled_after_partial(&mut self, order_id: OrderId);
    pub fn chain(&self, order_id: OrderId) -> Option<&[FillRecord]>;
    pub fn clear(&mut self);
    pub fn clear_for_instrument(&mut self, instrument: &Instrument);
}

pub struct L1MatchingEngine {
    books: HashMap<Instrument, L1Book>,
    trade_sequence: AtomicU64,
    tracker: PartialFillTracker,  // 0.8.0 Phase 3.2 新增
}
```

**关键设计点**:
- `FillRecord` 与 `Order` 解耦,`#[serde(default)]` 向前兼容 0.7.x 数据
- `FillState` 状态机 `Active → 终态` 单向,终态不可互转
- tracker 跨 instrument 共享(全局 fill 链),`L1Book::match_against_*` 关联函数加 `tracker` + `taker_instrument` 参数
- `track_fill` 同时记录 taker + maker 所属 instrument,确保 `clear_for_instrument` 能清空纯 maker 链
- 撮合循环内 maker 全成 → `mark_filled(maker_id)`;`submit` taker 全成 → `mark_filled(taker.id)`;`cancel` → `mark_cancelled_after_partial(order_id)`
- `clear_book` / `clear_book_for` 同步清 tracker

**性能**(A3.0 bench `--quick`):
- L1 submit:533ns → 1040ns(退化 1.95x)
- L3 single asset:678ns → 1206ns(退化 1.78x)
- L3 / L2 比:1206/1122 = **1.07x**(在 2x perf gate 内,远低于 1.17x baseline 退化)
- 退化来源:每笔 fill 多 2 次 `FillRecord::new` + 2 次 `HashMap::entry().or_default().push()` + 1 次 `is_filled()`。A3.x(Arena / SoA)可压实,A1.1 优先正确性。

**验收**:
- [x] `FillRecord` + `FillState` 在 axon-core 定义,`#[serde(default)]` 兼容
- [x] `PartialFillTracker` 在 axon-backtest 实现,`track_fill` / `mark_filled` / `mark_cancelled_after_partial` / `chain` / `clear` / `clear_for_instrument` 全 API
- [x] `L1MatchingEngine` 集成 tracker,`L1Book::match_against_*` 关联函数签名前向兼容(加 tracker + taker_instrument 参数)
- [x] `submit` / `cancel` / `clear_book` / `clear_book_for` 同步驱动 tracker
- [x] tracker 16 个单元测试 + L1 集成 9 个 e2e 测试
- [x] `cargo test --workspace` 0 failed(305 axon-backtest lib test,无回归)
- [x] `cargo clippy --workspace --all-targets -- -D warnings` 干净
- [x] `RUSTDOCFLAGS="-D warnings" cargo doc --workspace --no-deps` 干净
- [x] `cargo fmt -p axon-backtest -p axon-core` 干净
- [x] A3.0 bench `--quick` 跑通,L3/L2 = 1.07x ≤ 2x gate

**依赖**:`A2.1` 已在 2026-07-21 完成 ✅
**风险**:L1 内部细节(hidden)需要小心解耦,避免破坏 L1 单 book 路径 → 25 个原 L1 测试全过,无回归 ✅

**为什么本阶段只做 tracker + L1 集成(不重写 L3CoreMatchingEngine)**:

- 0.9.0 评估:如果 L3 多 leg 路由(A1.2)需要更激进的"价位簿重做"(SoA),再启动 L3CoreMatchingEngine 重写

```rust
pub struct OrderArena {
    /// 固定大小 chunk 池(O(1) 分配)

impl OrderArena {
    pub fn alloc(&mut self) -> OrderHandle;
    pub fn get(&self, h: OrderHandle) -> &Order;
    pub fn get_mut(&mut self, h: OrderHandle) -> &mut Order;
    pub fn free(&mut self, h: OrderHandle);
}
```
