> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 把 spot+perp 多 leg 抽象从"backtest 路径"扩展到"全栈统一",并把 0.5.0 留下的手动 `rebalance_to_target` / 手动 `push_funding` 自动化。涵盖 6 个 phase,L1+ ~2100 LOC。

**Architecture:** 6 phase 严格按依赖关系递进。P1+P2 是回测路径自动化(不依赖其他);P3 L3 撮合 + P4 streaming 是 engine 全面 Instrument 化(BREAKING);P5 OMS 加 instrument 字段(保持 Decimal 并行);P6 跨 leg 风险(新 LegPair 类型 + per-leg VaR)。每个 phase 一个 commit,`make test-ci` 验证后合入 0.6.0 分支。

**Tech Stack:** Rust 1.96.0、cargo、clippy、cargo-fmt、PyO3 0.28、rust_decimal::Decimal、parking_lot、tracing、serde

```rust
/// 0.6.0 新增(Phase 2):funding 8h 自动调度配置
///
/// 引擎在每根 bar 末(`begin_bar` 收尾)检查 schedule:若
/// `last_funding_ts[instrument] + interval_ns <= bar_ts` 合成 `FundingEvent`
/// 推 queue。`mark_aware = true` 时用 `mark_cache[instrument]`,否则用
/// 当前 `fallback_mark`(通常是 fill_price)。
///
/// 真实交易所(Binance/OKX)典型 8h 一次,UTC 00:00 / 08:00 / 16:00 整点;
/// interval_ns 是相对间隔,不强制对齐到整点。
#[derive(Debug, Clone, PartialEq, Serialize, Deserialize)]
pub struct FundingSchedule {
    /// 永续合约品种
    pub instrument: Instrument,
    /// 结算间隔(ns)。典型 8h = 28_800_000_000_000
    pub interval_ns: i64,
    /// 资金费率(正=long 付,负=long 收)
    pub fixed_rate: f64,
    /// `true`:用 `mark_cache[instrument]`;`false`:用 `fallback_mark`
    pub mark_aware: bool,
}

impl FundingSchedule {
    /// 构造 8h fixed-rate schedule(回测常用)
    pub fn fixed_8h(instrument: Instrument, rate: f64) -> Self {
        Self {
            instrument,
            interval_ns: 8 * 3600 * 1_000_000_000,
            fixed_rate: rate,
            mark_aware: true,
        }
    }

    /// 计算下一次 funding 时间戳
    #[inline]
    pub fn next_funding_ts(&self, prev_ts: Timestamp) -> Timestamp {
        Timestamp::from_nanos(prev_ts.as_nanos() + self.interval_ns)
    }
}
```

    /// `last_funding_ts[instrument] + interval_ns <= bar_ts` 合成

    /// - `interval_ns`: 结算间隔(ns)。典型 8h = 28_800_000_000_000

    #[pyo3(signature = (instrument, interval_ns, fixed_rate, mark_aware = true))]
    #[allow(clippy::too_many_arguments)]
    fn with_funding_schedule(
        &mut self,
        py: Python<'_>,
        instrument: &Bound<'_, PyAny>,
        interval_ns: i64,
        fixed_rate: f64,
        mark_aware: bool,
    ) -> PyResult<()> {

        let schedule = FundingSchedule {
            instrument: inst,
            interval_ns,
            fixed_rate,
            mark_aware,
        };
        self.inner.with_funding_schedule(schedule);
        Ok(())
    }

//! 0.6.0 新增:LegPair — 跨 leg 配对抽象
//!
//! 表示"同标的 spot + perp 对冲对",用于:
//! - 跨 leg 风险约束(perp 净敞口 = spot qty - perp qty × hedge_ratio)
//! - 跨 leg VaR 计算(covariance + 个体 VaR 聚合)

use serde::{Deserialize, Serialize};

use super::instrument::Instrument;

/// Spot + perp 对冲对
///
/// # 0.6.0 新增
///
/// 替代 axon-backtest::matching::l3::CrossPair 的 spot/perp 部分。
/// `hedge_ratio` 通常 1.0(BTC spot + BTC perp 1:1 对冲),但允许调整
/// (e.g. ETH/BTC pair 0.06 ratio + perp ETH 1:1)。
///
/// `axon-risk` 通过 `LegPair` 算净敞口,不直接依赖 `axon-backtest`。

    /// 现货 leg
    pub spot: Instrument,
    /// 永续合约 leg
    pub perp: Instrument,
    /// 对冲比率(perp qty = spot qty × hedge_ratio 时为 delta 中性)
    pub hedge_ratio: f64,
}

    /// 构造 spot+perp 1:1 对冲对(常用)
    pub fn new(spot: Instrument, perp: Instrument) -> Self {
        Self {
            spot,
            perp,
            hedge_ratio: 1.0,
        }
    }

    /// 构造带自定义 hedge_ratio 的对冲对
    pub fn with_ratio(spot: Instrument, perp: Instrument, hedge_ratio: f64) -> Self {
        Self {
            spot,
            perp,
            hedge_ratio,
        }
    }
}

//! 0.6.0 新增(Phase 6):跨 leg 净敞口检查
//!
//! 对每 LegPair:`net_delta = qty(spot) - qty(perp) * hedge_ratio`。
//! 若 |net_delta| > config.max_pair_drift → 拒 rebalance 单。

use axon_core::portfolio::Portfolio;
use axon_core::types::LegPair;

use crate::config::RiskConfig;
use crate::error::{RiskReason, RiskResult};

#[inline]
pub fn check_pair_net_exposure(
    portfolio: &Portfolio,
    pair: &LegPair,
    config: &RiskConfig,
) -> RiskResult {
    let spot_qty = portfolio
        .position_by_instrument(&pair.spot)
        .map(|p| p.quantity.as_f64())
        .unwrap_or(0.0);
    let perp_qty = portfolio
        .position_by_instrument(&pair.perp)
        .map(|p| p.quantity.as_f64())
        .unwrap_or(0.0);
    let net_delta = spot_qty - perp_qty * pair.hedge_ratio;
    if net_delta.abs() > config.max_pair_drift {
        return RiskResult::Reject(RiskReason::PairDriftExceeded {
            pair_label: format!("{:?}/{:?}", pair.spot, pair.perp),
            net_delta,
            limit: config.max_pair_drift,
        });
    }
    RiskResult::Allow
}
```

//! 0.6.0 新增(Phase 6):stress test 假设性 NAV 计算
//!
//! 对每 StressScenario 应用 price shock,算 hypothetical NAV。

use axon_core::portfolio::Portfolio;
use axon_core::types::Instrument;
use serde::{Deserialize, Serialize};

use crate::config::StressScenario;

#[derive(Debug, Clone, PartialEq, Serialize, Deserialize)]
pub struct StressTestResult {
    pub scenarios: Vec<StressScenarioResult>,
}

#[derive(Debug, Clone, PartialEq, Serialize, Deserialize)]
pub struct StressScenarioResult {
    pub scenario_name: String,
    pub original_nav: f64,
    pub shocked_nav: f64,
    pub nav_drop_pct: f64,
}

pub fn apply_stress_scenarios(
    portfolio: &Portfolio,
    scenarios: &[StressScenario],
    mark_prices: &std::collections::HashMap<Instrument, f64>,
) -> StressTestResult {
    // 计算原始 NAV
    let original_nav = compute_nav(portfolio, mark_prices);

    let mut results = Vec::new();
    for scenario in scenarios {
        let shocked_marks = apply_shock(mark_prices, scenario);
        let shocked_nav = compute_nav(portfolio, &shocked_marks);
        let drop_pct = if original_nav > 0.0 {
            (original_nav - shocked_nav) / original_nav
        } else {
            0.0
        };
        results.push(StressScenarioResult {
            scenario_name: scenario.name.clone(),
            original_nav,
            shocked_nav,
            nav_drop_pct: drop_pct,
        });
    }
    StressTestResult { scenarios: results }
}

fn compute_nav(
    portfolio: &Portfolio,
    mark_prices: &std::collections::HashMap<Instrument, f64>,
) -> f64 {
    let position_value: f64 = portfolio
        .positions()
        .iter()
        .map(|(inst, pos)| {
            let mark = mark_prices.get(inst).copied().unwrap_or(0.0);
            pos.quantity.as_f64() * mark
        })
        .sum();
    portfolio.cash_total() + position_value
}

fn apply_shock(
    marks: &std::collections::HashMap<Instrument, f64>,
    scenario: &StressScenario,
) -> std::collections::HashMap<Instrument, f64> {
    marks
        .iter()
        .map(|(inst, p)| {
            let shocked = if inst == &scenario.instrument {
                p * (1.0 + scenario.price_shock_pct)
            } else {
                *p
            };
            (inst.clone(), shocked)
        })
        .collect()
}
```

需要 `Portfolio::cash_total()` 存在(若没有则需在 axon-core 加)。
