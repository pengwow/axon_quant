## 1. 背景与目标

0.9.0 完成了 RL/HPO 训练生产化(`BacktestEnv` / `OnnxPolicyStrategy` / `RLHPOSweeper`),
但 LLM 智能体交易仅停留在 `docs/{en,zh}/user-guide/llm-trading/` 5 篇文章 + `crates/axon-llm`
中的局部实现,缺少**端到端 Python 跑通 + 训练数据可复现**的能力。

0.10.0 目标:**让 Python 端能用 ReAct agent + BacktestEnv 在 50 bar 上跑交易决策,
产出可复现 trajectory JSON**,为后续 SFT / RLHF / 数据集构建提供数据源。

### 1.1 关键决策(用户已确认)

| 决策项 | 选定 | 原因 |
|--------|------|------|
| Provider | **Mock + Ollama** | Mock 零成本跑 CI;Ollama 本地 $0 跑真模型 |
| 框架 | **ReAct** (Thought-Action-Observation) | 适合交易循环 |
| Memory | **5-turn sliding window** | 控制 prompt 长度,降低成本 |
| Trajectory | **全程 JSON 落盘** | 可复现,可喂 SFT/RLHF |
| Demo | **CLI 50 bar** (无 Web UI) | 最小验证路径 |
| Multi-agent / Web UI / Multi-LLM ensemble | ❌ **留 0.11.0** | 单次 spec 范围可控 |

### 1.2 现状审计

| 模块 | Rust 端 | Python 端 | 0.10.0 增量 |

### 2.2 数据流(运行时)

```
User CLI: python run_50bar.py --provider ollama --model llama3.2:3b
    │
    ▼
[make_backend("ollama")] → PyLLMBackend
    │
    ▼
[PyReActAgent.new(backend, max_memory=5, tools=[...])]
    │
    ▼
for bar in bars[0..50]:
    │
    ├─→ BarContext{ts, open, high, low, close, volume} → LLMMessage (user)
    │
    ├─→ backend.chat([history..., BarContext]) → LLMResponse
    │     (text + optional tool_call)
    │
    ├─→ if tool_call:
    │     PyPlaceOrderTool.execute(args)
    │     PyBacktestTradingBackend.place_order(args)
    │     → OrderAck → observation string
    │
    ├─→ TrajectoryRecorder.record({
    │     bar_id, ts, thought, action, observation, reward, cum_pnl
    │   })
    │
    └─→ if finish_bar called OR no tool_call:
         advance bar; continue

end → TrajectoryRecorder.flush() → artifacts/trajectory_<run_id>.json
    │
    ▼
[Reporter] print summary: total_pnl, trades, wall_time_s, cost_usd
```

### E4 — Trajectory 落盘(5 task)

**目标**:每次 ReAct step 记录一条轨迹,结束后落盘 JSON,符合 schema v0.10.0。

**关键文件**:
- `crates/axon-llm/src/trading/trajectory.rs` (~250 行):
  ```rust
  pub struct TrajectoryRecorder {
      run_id: Uuid,
      bars: Vec<TrajectoryBar>,
      start_ts: i64,
  }

  pub struct TrajectoryBar {
      bar_id: u64,
      ts: i64,
      thought: String,
      action: Option<ToolCall>,
      observation: Option<String>,
      reward: f64,
      cum_pnl: f64,
  }

  pub struct TrajectorySummary {
      total_pnl: f64,
      trades: u64,
      final_position: f64,
      wall_time_s: f64,
      cost_usd: f64,
  }

  impl TrajectoryRecorder {
      pub fn new(seed: u64, instrument: String, provider: String, model: String) -> Self;
      pub fn record(&mut self, bar: TrajectoryBar);
      pub fn flush(&self, path: &Path) -> Result<()>;
      pub fn replay(path: &Path) -> Result<Self>;  // 重放用
  }
  ```
- `crates/axon-llm/src/python/trajectory.rs` (~120 行): `PyTrajectoryRecorder` pyclass

### 4.3 错误处理

| 错误 | HTTP/code | 处理 |
|------|-----------|------|
| Ollama 连接失败 | `ConnectionError` | 立即返回 + 错误消息,不让 demo 跑飞 |
| Ollama 模型不存在 | `ModelNotFound` | 提示 `ollama pull <model>` |
| Tool call 参数错误 | `ToolError::InvalidArgs` | observation 返回错误字符串,ReAct 继续 |
| Trajectory 落盘失败 | `IOError` | 退出码 2,打印 stack |
| LLM 返回非法 JSON | `serde_json::Error` | 视为 finish_bar 触发,bar 结束 |
